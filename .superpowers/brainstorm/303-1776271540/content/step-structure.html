How many steps should the new wizard have?

Both start with scraping. The difference is whether events get their own review step during onboarding.

A
Option A — 5 steps
Lean wizard

Events from the scrape go straight into draft events for later review. Onboarding stays focused.

⭐ Recommended
⚡
1 · Smart Import Redesigned
Enter website URL → one-click scrape → review pre-filled org info (name, description, contact, logo)
🏷
2 · Services Redesigned
Visual category picker — auto-suggestions highlighted from scrape. Add service details inline.
📋
3 · Referral Setup
Delivery channels + custom form fields. Simplified card-based builder.
🌐
4 · Website Config
Confirm events & news page URLs. Scraped events auto-queued as drafts — they'll appear in your events dashboard.
✓
5 · Review & Publish
Clean summary of everything. One button to go live.
B
Option B — 6 steps
With inline event review

Scraped events get their own step so org managers confirm what gets imported before going live.

⚡
1 · Smart Import Redesigned
Enter website URL → scrape → review pre-filled org info
🏷
2 · Services Redesigned
Visual category picker with auto-suggestions from scrape
📅
3 · Events New
Scraped events shown as cards — tick to import, untick to skip. Dates, titles editable before import.
📋
4 · Referral Setup
Delivery channels + custom form fields
🌐
5 · Website Config
Confirm scraping URLs for ongoing weekly sync
✓
6 · Review & Publish
Summary + go live
Note: Either way, scraped events are saved as unpublished drafts — the org manager reviews them in their Events dashboard before they go live. Option B just shows this inline during onboarding.
